# [HAN.HG] 한홍구 개인 홈페이지 — Personal Universe

건설엔지니어이자 경관 디자인 실무자인 **한홍구**님을 위한 미래지향적(Futuristic) 컨셉의 개인 홈페이지입니다.
유럽 6개국 여행 기록, 무음주·마라톤·등산 라이프스타일, 그리고 경관 디자이너의 시선을 담은 비주얼 아카이브를 소개합니다.

---

## 🎯 프로젝트 목표

- 여행(이탈리아, 크로아티아, 슬로베니아, 터키, 프랑스, 스페인)과 건강한 라이프스타일(무음주, 마라톤, 등산)을 중심으로 한 개인 브랜딩 페이지 구축
- 경관 디자인 업무 경력에 어울리는 **미래지향적·구조적 다크 테마** UI 적용
- 정적 사이트 환경에서 구현 가능한 인터랙션(파티클 배경, 스크롤 애니메이션, 차트, 라이트박스)으로 몰입감 있는 경험 제공

---

## ✅ 현재 구현된 기능

### 디자인 & 비주얼
- **다크 + 네온 테마**: 사이언(#00e5ff) · 마젠타(#ff2ecb) · 바이올렛 그라디언트 기반의 미래지향적 팔레트
- **파티클 네트워크 배경**: Canvas 기반 실시간 파티클 연결 애니메이션 (전 페이지 고정 배경)
- **스캔라인 오버레이**: SF 인터페이스 느낌의 미세한 스캔라인 효과
- **프리로더**: 로딩 시 회전 링 애니메이션
- **Orbitron / Rajdhani / Noto Sans KR** 폰트 조합으로 테크니컬 + 가독성 확보

### 섹션 구성
1. **HOME (히어로)**: 이름, 슬로건, 핵심 통계(방문국 6개국, 나이 53세, 음주 0회, 건강 몰입도 100%) 카운트업 애니메이션
2. **PROFILE (자기소개)**: 궤도(Orbit) 형태의 비주얼 아이콘, 직업/나이/여행지역/라이프스타일 정보 카드
3. **TRAVEL LOG (여행 기록)**: 이탈리아·크로아티아·슬로베니아·터키·프랑스·스페인 6개국 카드형 갤러리
4. **LIFESTYLE (건강)**: 마라톤·등산·무음주 카드 + Chart.js 레이더 차트로 "라이프 밸런스 지수" 시각화
5. **GALLERY (비주얼 아카이브)**: 8장 이미지 그리드 + 라이트박스(확대보기) 기능
6. **CONTACT**: 이메일/지역/키워드 카드 + 클라이언트 사이드 데모 문의 폼

### 인터랙션
- 스크롤 시 네비게이션 바 활성화 및 현재 섹션 하이라이트
- Intersection Observer 기반 스크롤 reveal 애니메이션
- 모바일 햄버거 메뉴
- 맨 위로 스크롤 버튼
- 갤러리 라이트박스 (클릭 확대, ESC/배경 클릭으로 닫기)
- 히어로 통계 숫자 카운트업 애니메이션

## 🗂️ 파일 구조 & 진입 경로

```
index.html          # 메인 페이지 (전체 단일 페이지 구성)
css/style.css        # 미래지향적 다크+네온 테마 스타일
js/main.js           # 파티클 배경, 네비게이션, 애니메이션, 차트, 라이트박스 로직
README.md            # 프로젝트 문서
```

### 페이지 내 앵커(섹션) 경로
- `index.html#home` — 히어로 섹션
- `index.html#about` — 프로필 소개
- `index.html#travel` — 여행 기록 (6개국)
- `index.html#lifestyle` — 건강 라이프스타일
- `index.html#gallery` — 비주얼 갤러리
- `index.html#contact` — 연락처 / 문의 폼(데모)

## 🧩 사용 기술 스택

- **HTML5 / CSS3 / Vanilla JavaScript (ES6+)**
- **Chart.js** — 라이프 밸런스 레이더 차트
- **Font Awesome 6.4.0** — 아이콘
- **Google Fonts**: Orbitron, Rajdhani, Noto Sans KR
- 이미지: `image_search` 툴을 통해 확보한 CC/PD 라이선스 이미지 사용 (이탈리아, 크로아티아, 슬로베니아, 터키, 프랑스, 스페인, 마라톤, 등산)

> 본 프로젝트는 데이터베이스나 서버 저장이 필요 없는 **완전 정적 사이트**로 구성되어 있으며, RESTful Table API는 현재 사용하지 않았습니다.

---

## 🚧 아직 구현되지 않은 기능

- 문의 폼의 실제 이메일 전송 기능 (현재는 클라이언트 데모용, 서버/이메일 서비스 연동 필요)
- 여행 기록별 상세 페이지(사진 다수, 여행기 텍스트)
- 실제 마라톤 기록/등산 기록 데이터 연동 (Strava API 등은 인증이 필요해 정적 사이트에서 직접 연동 불가)
- 다국어 지원 (영문 버전)
- 실제 소셜미디어 링크 연결 (현재는 자리 표시자 `#`)
- 방문자 카운터, 댓글/방명록 기능 (필요 시 RESTful Table API로 구현 가능)
